# profilesRec

Attacking the weakness of a text-encoder sequential recommender with an LLM-distilled
user-profile signal.

## Setup

We start from a text-only Sentence-Transformer (Qwen3-Embedding-0.6B) trained as a
two-tower sequential recommender on Amazon Beauty (`train.py`). On the test split with
`filter_seen_items=True` the baseline reaches:

- R@10  ≈ 0.0949
- NDCG@10 ≈ 0.0486

Per-cohort slicing (`evaluate_slices.py`) shows the encoder struggles on short-history
prefixes and long-tail targets — that is the weakness we attack.

## The attack

Per-user free-text profiles are generated by an LLM from the user's history (the v6cf
profile cache; see `generate_profiles.py`). At inference we encode the profile string
with the same Sentence-Transformer to get `p_i`, and combine it with the text-only
query embedding `q_i` through a learned **gated profile fusion head** (`fusion.py`,
`GatedProfileFusionHead`):

```
fused_i = normalize(q_i) + σ(gate(features_i)) · proj(p_i) · has_profile_i
```

The gate is trained with InfoNCE + an **oracle-supervised** BCE auxiliary loss on
`train_fusion.py`. Two oracle objectives are supported (CLI flag
`--gate-aux-objective`):

| objective       | label rule                                                         | what it trains for |
|-----------------|--------------------------------------------------------------------|--------------------|
| `any_uplift`    | `y=1 iff rank_fused(gold) < rank_text(gold)`                       | soft re-ranking — improves NDCG@10 |
| `r10_recovery`  | `y=1 iff rank_text(gold) > 10 AND rank_fused(gold) ≤ 10`           | top-10 boundary crossings — targets Recall@10 |

Late-fusion at `--gate-aux-alpha` (default 0.3) defines the "what would happen at
training time if we mixed text and profile" pseudo-label. The `r10_recovery` objective
yields a sparse positive class, so pair it with `--gate-aux-pos-weight balanced`.

## Evaluation

- `evaluate_slices.py` — baseline (text-only) per-cohort metrics, writes
  `per_example.parquet` for downstream joins.
- `slice_fusion.py` — per-cohort fusion vs zero-profile vs baseline, with
  `--filter-seen-items` to match the trainer's eval regime.
- `analyze_oracle_slices.py` — paired bootstrap 95% CI, McNemar, transition tables,
  and history × target-popularity pivots; consumes the parquet from `slice_fusion.py`,
  CPU-only, no re-encoding.

Slices reported:
- Full dataset
- Cold users (history 3–5)
- Long history (21+)
- Popular targets (Q5, train freq > 59)
- Long-tail targets (Q1, train freq ≤ 7)
- Cold × Popular  /  Cold × Long-tail
- Long-hist × Popular  /  Long-hist × Long-tail
- has_profile warm/cold
- Gate-tertile cohorts (warm only)

## Pipeline

1. Train the text-only seqrec backbone with `train.py` (Qwen3-Embedding-0.6B,
   `--enrich-text-input`, `--filter-seen-items`, Beauty). Produces
   `outputs/seqrec_qwen3_beauty/best/`.
2. Generate LLM profile cache with `generate_profiles.py`. Produces a JSONL cache
   keyed by `profile_cache_key(history_item_ids, max_history)`.
3. Train the gated fusion head and run slice eval:
   - `run_eval_oracle.sh` — `any_uplift` objective (NDCG@10 lift).
   - `run_train_oracle_r10.sh` — `r10_recovery` objective (Recall@10 lift).
4. Read the per-cohort table in `<oracle_dir>/slices/oracle_vs_baseline.json` and the
   printed pivots in the SLURM log.

## File layout

```
config.py                    shared CLI parsers (train, fusion, profile generation)
data.py                      Amazon-2018 loader, build_examples, eval catalog
evaluation.py                encode_documents/queries, recall_and_ndcg_at_ks
profiles.py                  profile cache load + key (cache prefix hash)
fusion.py                    FusionHead, GatedProfileFusionHead, gate MLP

train.py                     seqrec backbone trainer
train_fusion.py              gated fusion head trainer (oracle BCE auxiliary)
generate_profiles.py         LLM profile generation

evaluate_slices.py           baseline per-cohort eval
slice_fusion.py              fusion per-cohort eval (joins baseline parquet)
analyze_oracle_slices.py     bootstrap CIs / McNemar / cohort pivots

run_eval_oracle.sh           SLURM: baseline regen + any_uplift oracle + slice eval
run_train_oracle_r10.sh      SLURM: r10_recovery oracle + slice eval
utils.py                     save_json, set_seed, truncate_words, parse_timestamp
```

## Environment

The SLURM scripts expect `conda activate my_env` and `HF_TOKEN` set in the calling
shell (do not commit tokens). Python deps are the usual ML stack: `torch`,
`sentence-transformers`, `transformers`, `pandas`, `numpy`, `sklearn`. Profile
generation uses an LLM via the relevant provider SDK.
